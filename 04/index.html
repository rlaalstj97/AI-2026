Web Ecosystem Blueprint

웹(Web) vs 인터넷(Internet)

많은 사람들이 혼동하지만, 둘은 명확히 다릅니다.
인터넷이 전 세계의 컴퓨터를 연결하는 '하드웨어 도로'라면, 웹은 그 도로 위를 달리는 '정보 서비스'입니다.

🔗

URL

고유한 자원의 위치 식별 체계

📡

HTTP

자원을 주고받기 위한 통신 약속

📝

HTML

정보의 구조와 링크를 정의하는 언어

🌐

Request & Response 흐름

1. URL 입력
2. DNS 도메인 해석
3. TCP 연결 수립 (Handshake)
4. HTTP 요청 전송
5. 서버 로직 처리
6. 브라우저 렌더링 파이프라인 가동

웹의 세대별 진화

1989 - 2004

정적 정보의 시대

사용자는 정보를 읽기만 할 수 있었습니다. 기업이나 기관이 제공하는 텍스트 위주의 페이지가 주를 이루었습니다.

  • ✅ HTML, CGI 중심
  • ✅ 단순한 텍스트와 하이퍼링크
  • ✅ 브라우저 전쟁 (IE vs Netscape)

"문서를 웹에 올리는 것 자체가 혁명이던 시절"

2004 - 현재

참여와 공유의 플랫폼

사용자가 직접 콘텐츠를 생성합니다. SNS, 유튜브, 클라우드 기술이 폭발적으로 성장했습니다.

  • ✅ AJAX, JSON 기반 비동기 통신
  • ✅ 스마트폰 보급과 모바일 퍼스트
  • ✅ 고성능 크롬(V8 엔진)의 등장

"소통하고 참여하는 소셜 네트워크 서비스"
